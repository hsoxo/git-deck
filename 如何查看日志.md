# 如何查看 Git GUI 日志

## 快速开始

### 1. 打开日志面板

**方法一：使用快捷键**
- Windows/Linux: `Ctrl + Shift + U`
- macOS: `Cmd + Shift + U`
- 然后在右上角下拉菜单选择 **"Git GUI"**

**方法二：使用命令**
- 按 `Ctrl + Shift + P` (macOS: `Cmd + Shift + P`)
- 输入 `Git GUI: Show Output`
- 回车

**方法三：使用菜单**
- 点击菜单 `View` → `Output`
- 在右上角下拉菜单选择 **"Git GUI"**

### 2. 启用详细日志

打开 VS Code 设置（`Ctrl + ,`），搜索 `gitGui.logLevel`，设置为 `debug`

或者直接编辑 `settings.json`：
```json
{
  "gitGui.logLevel": "debug"
}
```

### 3. 重新加载窗口

- 按 `Ctrl + Shift + P` (macOS: `Cmd + Shift + P`)
- 输入 `Developer: Reload Window`
- 回车

### 4. 打开 Git GUI

- 按 `Ctrl + Shift + P` (macOS: `Cmd + Shift + P`)
- 输入 `Git GUI: Open`
- 回车

### 5. 查看日志

切换到 Output 面板（如果还没打开），你会看到详细的日志输出。

## 日志内容说明

日志会显示：

- ✅ 扩展启动信息
- ✅ Git 仓库路径
- ✅ Webview 加载过程
- ✅ 前后端通信（RPC 调用）
- ✅ Git 操作执行情况
- ✅ 错误和警告信息
- ✅ 性能计时信息

## 查看前端日志

除了 Output 面板，你还可以查看 Webview 的浏览器控制台：

1. 按 `Ctrl + Shift + P` (macOS: `Cmd + Shift + P`)
2. 输入 `Developer: Open Webview Developer Tools`
3. 选择 `Git GUI`
4. 查看 Console 标签页

## 常见问题

### Q: 看不到 "Git GUI" 选项？
A: 确保扩展已激活，尝试先打开 Git GUI，然后再查看 Output 面板。

### Q: 日志太少，看不到详细信息？
A: 确认 `gitGui.logLevel` 设置为 `debug`，然后重新加载窗口。

### Q: GUI 一直显示 Loading，日志里有什么线索？
A: 查找以下关键信息：
- `Webview panel created successfully` - 面板是否创建成功
- `Script URI:` - 资源路径是否正确
- `Received message from webview` - 前后端是否通信
- `RPC call: git.getStatus` - Git 操作是否执行
- 任何 `[ERROR]` 标记的错误信息
